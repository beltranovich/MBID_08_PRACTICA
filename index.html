Análisis EDA

El análisis exploratorio de datos (EDA, por sus siglas en inglés) es una etapa fundamental en cualquier proyecto de análisis de datos. Consiste en explorar y comprender los datos disponibles antes de aplicar cualquier modelo o realizar inferencias. El objetivo principal del EDA es descubrir patrones, tendencias, relaciones y características destacadas de los datos.

En nuestro proyecto de análisis de accidentes en la ciudad de Nueva York, utilizaremos técnicas de EDA para explorar y visualizar los datos. Utilizaremos un notebook de Python colaborativo que nos permitirá ejecutar y documentar nuestro análisis paso a paso. El notebook nos brinda la flexibilidad de cargar y manipular los datos, realizar transformaciones y cálculos, y crear visualizaciones interactivas.
Nuestro objetivo principal en este proyecto es identificar patrones en los accidentes de tránsito, entender cómo han fluctuado a lo largo del tiempo, analizar la distribución de accidentes por hora del día y día de la semana, examinar las causas más comunes de los accidentes y explorar las diferencias entre los distritos de la ciudad.

Notebook Python

En un notebook de Python desde Google Colab, tenemos varias posibilidades para realizar análisis exploratorio de datos (EDA).
Algunas de las herramientas que usaremos en la practica son:

- Librerías de Visualización: Matplotlib, Seaborn y Plotly para crear gráficas y visualizaciones interactivas.
- Librerías para Análisis de Datos: Pandas y NumPy para realizar manipulación y análisis de datos, como filtrar, agrupar, calcular estadísticas descriptivas y hacer operaciones con matrices.
- Librerias de Graficos Georeferenciados: Folium te permite crear mapas interactivos utilizando datos geoespaciales.

Con estas herramientas podemos cargar y explorar datos de manera eficiente, así como visualizar patrones y descubrir relaciones entre las diferentes columnas.
Además, el entorno colaborativo del notebook de Python nos permite realizar un análisis EDA paso a paso, brindando la oportunidad de agregar información explicativa en formato Markdown antes de ejecutar las celdas de código.
